## ytgrab core: the resilient downloader

- ask: ytgrab, end to end, click-to-run (your settled option 1) — you said you would not do the terminal yourself, so there is none to do. A "ytgrab" entry is in your applications menu; clicking it the first time asks for the channel and the save folder with two dialogs (no file editing), then opens a terminal window showing the download running; clicking it again any time resumes — a lock means a second click never double-runs, and if a run is already live it just tells you. Underneath, the core (`ytgrab.py`) wraps yt-dlp: a whole channel oldest-first, ≤720p merged to mp4, exact resume across any stop/restart/crash (download-archive + partial continue), a polite randomised sleep so it is never rate-limited, and a free-space guard that stops with a message instead of crashing (a click resumes once space is freed). Two build slices — the core and the desktop shortcut + launcher — both built; this one card accepts the whole click-to-run experience.
- try: first install the two pieces this needs — `sudo apt install ffmpeg` (not installed here; yt-dlp needs it to merge picture+sound into mp4) and `pip install -U yt-dlp` (note the `-U`: the yt-dlp on this machine, 2024.04.09, is ~14 months old and can no longer read YouTube — in your two runs today it found nothing and the program wrongly logged "done"; that lie is now fixed, but only an up-to-date yt-dlp will actually download). Then open your applications menu and click "ytgrab" (a relogin may be needed for the menu to refresh). Paste a channel URL or @handle in the first dialog (a bare @handle now works — it used to be silently rejected), choose a save folder in the second — a terminal opens and starts downloading oldest video first, pausing politely between videos. Close the terminal (or log out, or reboot) to stop; click "ytgrab" again and it resumes where it left off, skipping what finished. Click it while it is running and it tells you it is already going rather than starting a second. If a run ever saves nothing it now says so plainly (wrong handle, or an outdated yt-dlp) instead of claiming success.
